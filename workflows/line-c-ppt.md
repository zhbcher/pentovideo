# 线C：PPT → 识图 → HTML → 视频（有/无口播稿双分支）

用户提供 PPT/PDF 文件，自动拆图、识图、对齐口播、配音、渲染。

## 有口播稿分支（9阶段）

```
阶段1：文档转换 → LibreOffice → PPTX/PDF → 逐页PNG
阶段2：OCR识图 → Mistral → 逐页提取文字 → 识别稿
阶段3：口播稿对齐 → 语义匹配 → ≥70%自动对齐，30-70%确认，<30%人工
阶段4：TTS配音 → 合并一段 → narration.mp3
阶段5：语音转字幕 → transcribe → transcript.json
阶段6：时间轴构建 → build_timeline.py
阶段7：写HTML → 图片背景+字幕 → lint → validate
阶段8：预览确认 → ★ 用户确认
阶段9：渲染+交付
```

## 无口播稿分支（9阶段）

```
阶段1-2：同上
阶段3：AI写口播稿 → voiceover.md → ★ 用户确认/修改
阶段4-9：同有口播稿分支
```

## 对齐规则

- 识别稿每页全文 vs 原稿每页全文 → 语义相似度匹配
- ≥70%：自动对齐，采用原稿对应页内容
- 30%-70%：报用户确认
- <30%：报用户决策
- **页数差 ≥2 不进对齐**：直接报用户
